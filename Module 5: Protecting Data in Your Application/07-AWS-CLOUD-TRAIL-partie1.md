# AWS CloudTrail - partie 1
AWS CloudTrail est un service d'Amazon Web Services (AWS) qui permet d'auditer, de surveiller et de suivre les activités de votre compte AWS. CloudTrail enregistre les actions effectuées par un utilisateur, un rôle ou un service AWS dans votre compte. Ces actions sont appelées événements CloudTrail. Les événements CloudTrail peuvent inclure des actions prises via la console AWS, les commandes AWS CLI, les SDK AWS et d'autres services AWS.

Les principales fonctionnalités d'AWS CloudTrail comprennent :

1. **Enregistrement des événements** : CloudTrail enregistre toutes les actions (événements) effectuées sur votre compte AWS, telles que les appels d'API, les actions de la console et les commandes CLI.

2. **Livraison des journaux** : Les journaux CloudTrail sont livrés à un bucket Amazon S3 de votre choix, vous permettant de stocker et d'analyser les événements à des fins d'audit et de conformité.

3. **Surveillance continue** : Vous pouvez configurer CloudTrail pour surveiller et détecter les activités suspectes ou non autorisées dans votre compte AWS en temps réel.

4. **Intégration avec d'autres services AWS** : CloudTrail peut être intégré à d'autres services AWS, comme Amazon CloudWatch, pour créer des alertes et des actions automatisées basées sur les événements enregistrés.

5. **Analyse des journaux** : Les journaux CloudTrail peuvent être analysés pour comprendre les tendances d'utilisation, identifier les anomalies et enquêter sur les incidents de sécurité.

En résumé, AWS CloudTrail est un outil essentiel pour assurer la transparence, la sécurité et la conformité des activités dans votre environnement AWS.
